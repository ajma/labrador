## Design Context

### Users
Casual-to-intermediate homelab enthusiasts. They self-host services like Nextcloud, Plex, Immich, and Vaultwarden on a home server, old desktop, or Raspberry Pi. They use Labrador in the evening from a desktop or laptop — they want to get in, deploy or check something, and get out. Comfortable with Docker but not DevOps professionals. They value clarity and directness over density and power-user options.

**Job to be done**: Deploy and monitor Compose stacks without fighting the tool.

### Brand Personality
**3 words**: simple, streamlined, purposeful.

The tone is that of a well-crafted indie tool — opinionated, honest about its scope, made for a specific person. Not a committee product, not enterprise software with a facelift. Confident without being loud.

**Anti-references**: Portainer (enterprise density, too many exposed knobs), Proxmox (dated admin-panel aesthetic), any generic dark dashboard with identical muted cards and one blue accent. If it looks like it shipped with a BI dashboard template, it's wrong.

### Aesthetic Direction

**Dark theme always** — used in the evening, home office context.

**Color**: Go beyond a single accent. The established blue (`#649ef5` / `#7db0ff`) stays as the primary interactive color. Status colors (green `#4ade80`, amber `#facc15`, red `#f87171`) should be vivid and immediately legible — not muted or tinted-away. Use color to communicate state, zone, and priority throughout the UI. Avoid the monochrome-with-one-accent trap — the interface should feel alive.

**Animation**: Motion is a first-class design material, not an afterthought. Use it for:
- State transitions (deploy status changes, running → stopped, etc.)
- Entrance animations on navigation and key content areas
- Micro-interactions on hover, click, and form feedback
- Progress and loading states that feel alive, not placeholder-y
Default easing: `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-expo). Duration ladder: 150ms micro, 250ms state, 400ms layout, 600ms entrance.

**Typography**: **Rubik** as the single font family. Slightly rounded geometric grotesque — readable at normal UI sizes (13–18px), anti-enterprise without being playful. Use weight contrast for hierarchy (400 body, 500 labels, 600 headings, 700 display). Monospace system stack for code/compose content only.

Font sizes use a 7-step semantic scale defined in `tailwind.config.ts`. Always use these classes — never `text-[Xpx]` arbitrary values:

| Class | Size | Use |
|-------|------|-----|
| `text-2xs` | 12px | Chips, table headers, uppercase labels |
| `text-xs` | 13px | Muted/secondary text |
| `text-sm` | 14px | Primary body text |
| `text-md` | 15px | Slightly larger body |
| `text-lg` | 18px | Section headings |
| `text-xl` | 22px | Larger headings |
| `text-2xl` | 27px | Page-level headings |

To scale all text globally, edit the rem values in `tailwind.config.ts` only — never touch individual files.

**Visual base**:
- Background: `#04070f` family (deep navy-black) — established, keep it.
- Border-radius: `rounded-xl` to `rounded-2xl` as the default range.
- Borders: ultra-low opacity whites (`rgba(255,255,255,0.06–0.12)`) for surfaces; blue-tinted on focus.
- Surfaces: layered dark with subtle elevation differences, not flat.

### Design Principles
1. **Simple over complete** — show what's needed for the task. Omit rather than collapse. The interface steps aside.
2. **Color as signal** — every color communicates something: state, priority, zone, or feedback. Never decoration for its own sake.
3. **Motion as feedback** — transitions confirm actions, entrances orient the user, micro-interactions acknowledge input. Motion earns its place or doesn't ship.
4. **Anti-enterprise** — personality over polish. If it looks like it was built by a vendor for IT procurement, it's wrong.
5. **Purposeful emptiness** — empty states teach. "Nothing here yet" is never acceptable copy.
